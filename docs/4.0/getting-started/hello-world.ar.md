# مرحباً بالعالم

سيرشدك هذا الدليل خطوة بخطوة خلال إنشاء مشروع Vapor جديد، وبنائه، وتشغيل الخادم.

إذا لم تكن قد ثبّت Swift أو Vapor Toolbox بعد، فاطّلع على قسم التثبيت.

- [التثبيت &rarr; macOS](../install/macos.md)
- [التثبيت &rarr; Linux](../install/linux.md)

!!! tip "نصيحة"
    يتطلب القالب المستخدم من قِبل Vapor Toolbox الإصدار Swift 6.0 أو أحدث

## مشروع جديد

الخطوة الأولى هي إنشاء مشروع Vapor جديد على جهازك. افتح الطرفية واستخدم أمر إنشاء المشروع الجديد من Toolbox. سيؤدي هذا إلى إنشاء مجلد جديد في الدليل الحالي يحتوي على المشروع.

```sh
vapor new hello -n
```

!!! tip "نصيحة"
    تمنحك الراية `-n` قالباً أساسياً بسيطاً عبر الإجابة بـ "لا" تلقائياً على جميع الأسئلة.

!!! tip "نصيحة"
    يمكنك أيضاً الحصول على أحدث قالب من GitHub بدون Vapor Toolbox عن طريق استنساخ [مستودع القالب](https://github.com/vapor/template-bare)

!!! tip "نصيحة"
    يستخدم Vapor والقالب الآن `async`/`await` بشكل افتراضي.
    إذا لم تتمكن من الترقية إلى macOS 12 و/أو كنت بحاجة إلى الاستمرار في استخدام `EventLoopFuture`s،
    فاستخدم الراية `--branch macos10-15`.

بمجرد انتهاء الأمر، انتقل إلى المجلد المُنشأ حديثاً:


```sh
cd hello
```

## البناء والتشغيل

### Xcode

أولاً، افتح المشروع في Xcode:

```sh
open Package.swift
```

سيبدأ تلقائياً في تنزيل اعتماديات Swift Package Manager. قد يستغرق هذا بعض الوقت في المرة الأولى التي تفتح فيها مشروعاً. عند اكتمال حل الاعتماديات، سيقوم Xcode بملء المخططات (schemes) المتاحة.

في أعلى النافذة، على يمين زرَّي التشغيل (Play) والإيقاف (Stop)، انقر على اسم مشروعك لتحديد مخطط (Scheme) المشروع، ثم اختر هدف تشغيل مناسباً—على الأرجح "My Mac". انقر على زر التشغيل لبناء مشروعك وتشغيله.

من المفترض أن ترى وحدة التحكم (Console) تظهر في أسفل نافذة Xcode.

```sh
[ INFO ] Server starting on http://127.0.0.1:8080
```

### Linux

على Linux وأنظمة التشغيل الأخرى (وحتى على macOS إذا لم ترغب في استخدام Xcode) يمكنك تحرير المشروع في محرّرك المفضّل، مثل Vim أو VSCode. راجع [Swift Server Guides](https://github.com/swift-server/guides/blob/main/docs/setup-and-ide-alternatives.md) للحصول على تفاصيل محدّثة حول إعداد بيئات تطوير متكاملة (IDEs) أخرى.

!!! tip "نصيحة"
    إذا كنت تستخدم VSCode كمحرّر للكود، فإننا نوصي بتثبيت امتداد Vapor الرسمي: [Vapor for VS Code](https://marketplace.visualstudio.com/items?itemName=Vapor.vapor-vscode).

لبناء مشروعك وتشغيله، نفّذ في الطرفية:

```sh
swift run
```

سيؤدي ذلك إلى بناء المشروع وتشغيله. في المرة الأولى التي تشغّل فيها هذا، سيستغرق بعض الوقت لجلب الاعتماديات وحلّها. بمجرد التشغيل، من المفترض أن ترى ما يلي في وحدة التحكم:

```sh
[ INFO ] Server starting on http://127.0.0.1:8080
```

## زيارة Localhost

افتح متصفح الويب، وقم بزيارة <a href="http://localhost:8080/hello" target="_blank">localhost:8080/hello</a> أو <a href="http://127.0.0.1:8080" target="_blank">http://127.0.0.1:8080</a>

من المفترض أن ترى الصفحة التالية.

```html
Hello, world!
```

تهانينا على إنشاء أول تطبيق Vapor خاص بك وبنائه وتشغيله! 🎉
